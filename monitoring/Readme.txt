Sistema de Monitoramento Linux.


O Sistema de Monitoramento Linux, contêm uma aplicação feita em linguagem Html, SGV, Javascript, Jquery, Css, que implementam um Mapa do Estado de São Paulo. 
Os ícones que contem no Mapa do Estado de São Paulo, podem ser identificados como por exemplo: Empresas, Computadores em redes, Postos ou Pontos e Outros.
O Funcionamento do Sistema consiste em Servidor de hospedagem para aplicação, e um Client baseado em Linux. Existe uma series de configurações feita em Bash Linux capaz de realizar o envio do arquivo via comando scp para o Servidor remoto.
A aplicação consiste em receber o arquivo do Client a cada 1 minuto que seguida faz a verificação e instantaneamente passa a processar em uma condição, que neste caso, se o arquivo foi recebido o sistema faz a monitoração do Client, caso contrário, sistema não monstra o Client sendo monitorado.
Existem diversas formas de configurações que podem ser implementadas no sistema, como avisos, ou até envios de e-mails para outros sistemas que podem ser criados de forma que obtenham um aviso do Client em casos de não responder.
Os arquivos criados podem ser modificados ou alterados de acordo com a necessidade, o mapa pode ser criado em diversos formatos de acordo com a topologia desejada, que neste projeto, o sistema foi baseado no mapa do Estado de São Paulo.




Monitoring system Linux.


The Monitoring System Linux, contain an application made in HTML language, SGV, Javascript, jQuery, CSS, to implement a Map of the State of São Paulo.
The icons that contains the map of the State of São Paulo, can be identified as: Business, Computer networks, stations or points and Others.
The system operation consists of hosting server for application and Client based on Linux. There is a series of settings made in Linux Bash able to perform sending the file via scp command to the remote server.
The application is to receive the Client file every 1 minute then makes the verification and instantly starts to process on one condition, which in this case, if the file was received the system does the Client monitoring, otherwise system no monster Client being monitored.
There are various forms of settings that can be implemented in the system, such as warnings, or even sends e-mails to other systems that can be created in order to obtain one Client notice in cases of non answer.
The files created can be modified or changed according to need, the map can be created in many different shapes according to the desired topology, which in this project, the system was based on the map of the State of São Paulo.
